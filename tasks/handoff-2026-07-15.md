## 已冻结的 Stage 21 决策

采用 validation-only：

```text
project root
  -> fixed handoff producer
  -> Stage 18 reference consumer
  -> safe validation result
```

继续禁止：

- descriptor argv 自动执行；
- HTML/JSON representation 自动读取；
- 浏览器、URL、socket、任意文件路径或 shell 字符串输入；
- 临时文件、artifact、ledger、draft 或其他持久写入；
- refresh、polling、live server、HTTP、WebSocket 或 background process；
- UI 事件直接推导执行权限；
- 真实 external adapter execution。

## Stage 22 进入条件

只有同时明确以下事项，才启动新实现：

1. 真实 representation consumer 与用户场景；
2. 用户显式选择 representation 类型和读取动作；
3. 固定 argv allowlist，不接受 descriptor 中任意新增命令；
4. project-root 路径归一化与越界拒绝；
5. stdout/HTML/JSON 上限、UTF-8 解析与脱敏；
6. timeout、取消、关闭与 no-retry 语义；
7. 内存展示、stdout、浏览器展示或受控 export 的明确选择；
8. 与 handoff/validation result 的 identity 关联；
9. no-write、no-network、no-service、no-adapter-execution 回归证据；
10. 若进入 UI，先冻结 UI action 与授权边界。

条件不完整时，应保持 unavailable / needs_input，不尝试读取。
