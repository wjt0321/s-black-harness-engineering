## Stage 26 输入契约

未来 Stage 27 filtered v3 候选输入：

```text
--representation snapshot-json
--envelope <validated-project-relative-path>
[--task-id <task-YYYYMMDD-NNN>] [--request-id <req-YYYYMMDD-NNN>]
```

- 至少一个 filter；
- 双参数使用 AND；
- filter 必须有 envelope；
- 每个 flag 最多一次；
- canonical ASCII exact match，不 trim、不折叠大小写；
- 不接受 list、wildcard、regex、query、sort、page 或 export。

## Filter 作用域与关系闭包

filter 只作用于完整验证后的：

- runs summaries；
- approvals summaries；
- artifacts summaries。

不包含 project overview/tasks、registry adapters、automation 或 raw envelope。

仅 task filter 时，先从 runs 建立：

```text
request_task[request_id] = task_id
```

再用 selected request ids 包含只有 request_id、缺少直接 task_id 的 response artifacts。仅 request filter 为 exact match；task+request 不一致时返回 ready 空视图，不降级或猜测。

## v3 Identity

计划版本：

- result：`control-plane/codex-desktop-snapshot-read/v3`
- reader：`codex-desktop-filtered-envelope-snapshot-json-reader/v3`
- payload：`control-plane/filtered-envelope-snapshot/v1`
- filter：`control-plane/envelope-snapshot-filter/v1`

identity：

```text
filter_id = sha256(canonical(filter))
view_id = sha256(canonical(filtered_payload_without_view_id))
```

filtered payload source 必须关联 Stage 24 `base_snapshot_id`、`scope_id` 与 `filter_id`。不得把 filtered payload 伪装成原始 `control-panel-snapshot/v1`，也不得复用 base snapshot id。

## 固定安全边界

- child argv 仍只有 handoff → reference consumer → snapshot，并只携带 validated envelope path；
- filter 在 base snapshot schema/source/identity/hash 与 envelope content drift 全部通过后，对内存安全 summaries 执行；
- reports 继续 unavailable，不自动调用 report generate；
- no query、no lineage expansion、no persistence/export、no HTML/browser、no network/service/write、no adapter execution。
