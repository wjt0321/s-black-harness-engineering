## Stage 22 固定动作

```text
explicit --representation snapshot-json
  -> fixed handoff producer
  -> Stage 18 reference consumer
  -> fixed snapshot producer
  -> schema/source/guarantees/identity/hash validation
  -> bounded JSON result
```

实际调用：

```bash
python tools/codex_desktop_snapshot_json_reader.py \
  --project-root . \
  --representation snapshot-json \
  --timeout-seconds 30 \
  --json
```

## 已冻结安全边界

- 只有 `snapshot-json`；无默认 representation；
- 三段 argv 全部由 reader 固定，不消费 descriptor argv；
- project root 必须包含 `pyproject.toml` 与 `agent_runtime/`；
- 每个子进程默认 30 秒、最大 60 秒；stdout/stderr 上限 1 MiB；固定禁用 Python bytecode cache 写入；
- strict UTF-8 JSON、duplicate-key 拒绝；
- snapshot id 必须匹配 handoff，并通过 canonical hash 重算；
- 输出不回显绝对 root、raw stderr、descriptor、argv 或环境变量；
- no retry、no write、no network、no service、no HTML、no adapter execution。

## Stage 23 进入条件

只有用户明确需要 runs / approvals / artifacts scoped snapshot 时，才设计 envelope 支持，并先冻结：

1. 用户显式授权；
2. project-relative envelope path allowlist；
3. 绝对路径、`..` 与 project-root 越界拒绝；
4. handoff、consumer、snapshot 三者 identity 关联；
5. 输入文件类型、大小与 secret scan 边界；
6. no-write/no-network/no-service 回归；
7. 不允许从 descriptor 或 UI 事件推导任意路径执行。

条件不完整时，保持 `envelope_required` / unavailable，不自动读取。
