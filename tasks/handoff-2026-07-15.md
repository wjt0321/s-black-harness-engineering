## Stage 25 决策

已评估三种方案：

1. 自动新增 `--task-id` / `--request-id`：拒绝；缺少消费者、组合语义与 identity 契约。
2. 新增通用 `--filter` / `--query`：拒绝；会引入 arbitrary query、字段探测与复杂度边界。
3. 保持单-envelope、无 filter v2，并冻结 host 展示契约：采用。

reader CLI 继续只有：

```text
--project-root
--representation snapshot-json
--envelope <allowlist project-relative path>
--timeout-seconds
--json
```

明确不提供：

```text
--task-id
--request-id
--filter
--query
--sort
--page
--export
```

## 宿主消费边界

允许：

```text
explicit user action
  -> fixed one-shot reader
  -> bounded stdout JSON
  -> status/schema/reader id validation
  -> process-local in-memory display
  -> discard on view close
```

禁止：

- 执行 descriptor/result/snapshot 中的 argv；
- 保存、cache、export、上传、发送或写回 representation；
- 将 representation 持久化为 Run/Event/Report/Artifact；
- 打开 HTML/browser、自动刷新、轮询或后台驻留；
- 网络、DB、auth/session、真实 adapter execution；
- host-local 任意筛选后冒充 reader 输出。

## 兼容性

- 无 envelope 继续返回 Stage 22 v1；
- 显式 envelope 继续返回 Stage 24 v2；
- v1/v2 schema、reader id、scope id、snapshot canonical hash 均未改变；
- reports 继续 request-scoped unavailable；
- no write、no network、no service、no HTML/browser、no adapter execution 保持不变。

## Stage 26 进入条件

只有具体消费者同时给出以下事实时才启动：

1. 结构化 task/request filter 的真实用户动作；
2. filter canonical form、空值、无匹配与组合语义；
3. filter identity 与 scope/snapshot/result identity 的绑定；
4. 确定性排序、结果数量和输出大小上限；
5. no arbitrary query/path、no persistence/export、no HTML/browser/network/write/execute 回归证据。

条件不完整时继续维持 Stage 25 冻结的单-envelope、无 filter v2。
