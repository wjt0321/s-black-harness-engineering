## 2. 当前唯一执行权限

唯一允许的真实 operation：

```text
actor = local-operator
adapter_id = shell-local
capability = git_status
operation = git_status
argv = ["git", "status", "--short", "--branch"]
shell = false
```

必须显式使用 `--commit`。CLI 不接受 command、argv、cwd、environment、executable path、binding path 或 actor override。

ready 只声明：

```text
no_write_contract = true
guard_evidence_passed = true
filesystem_write_proof = false
```

不得把该能力解释为通用 Git、通用 shell、完整 filesystem write protection 或任意 adapter execution。

## 3. 当前停止线

以下能力仍 unavailable：

1. POSIX executable image identity 与 process-group backend；
2. linked worktree、submodule、alternate object store 与 custom external excludes/attributes；
3. OS-enforced read-only filesystem proof；
4. approval-required adapter 的 canonical approval binding；
5. network adapter、消息发送、service、DB、UI、background worker；
6. 任意 argv、第二个 Git command 或第二个真实 operation。

Machine-local trust binding 需要 operator 独立 provision。Git binary、signer、file identity 或 sanitized PATH identity 漂移后，execution 必须 blocked，直到显式 `--replace --commit` rotation。

## 4. Stage 50 条件

下一阶段不自动增加 command。优先 design gate：

1. trust binding rotation 的 operator workflow 与失败恢复；
2. `awaiting_terminal` / open attempt 的诊断与恢复决策；
3. Windows Job accounting 与更强 no-orphan evidence；
4. terminal audit incomplete 时的安全 operator handling；
5. 若选择 POSIX，则单独设计 image identity、process-group containment 与等价 trust root。

任何第二个 operation、approval-required execution 或 stronger filesystem isolation 都需要独立设计与用户明确授权。
