## 本阶段完成内容

### Adapter execution envelope 设计

文件：

- `docs/12-adapter-execution-envelope.md`
- `adapters/execution-envelope.schema.json`
- `adapters/execution-envelope.examples.json`

定义四类 artifact：

- `adapter_request`
- `adapter_response`
- `approval_record`
- `execution_event`

### `adapter plan`

命令：

```bash
python -m agent_runtime.cli adapter plan --adapter github-cli --operation git_push --target origin/main
```

行为：

- 复用 `check action` preflight。
- 生成 envelope 草案，不执行 adapter。
- 需要授权时生成 `adapter_request`、`approval_record` 和 `approval_requested` execution event。
- 默认人类输出只展示 artifact 摘要；`--json` 输出完整 envelope。

### `adapter validate`

命令：

```bash
python -m agent_runtime.cli adapter validate --file adapters/execution-envelope.examples.json
```

行为：

- 校验 envelope JSON 是否符合 `adapters/execution-envelope.schema.json`。
- 校验 cross-artifact consistency：
  - `adapter_request.request_id` 唯一。
  - approval / response / event 的 `request_id` 必须引用存在的 request。
  - approval scope 必须与 request 的 task / adapter / operation / target 一致。
  - 需要 approval 的 request 必须存在 approval record。
  - `approval_requested` event 的 `approval_id` 必须引用存在的 approval record。

### `adapter inspect`

命令：

```bash
python -m agent_runtime.cli adapter inspect --file adapters/execution-envelope.examples.json
```

行为：

- 先执行 validate 语义。
- 输出 envelope 摘要：version、description、artifact counts、requests、approvals、responses、event counts、overall flags。
- 不打印完整 envelope 或 input payload。

### `adapter approval check`

命令：

```bash
python -m agent_runtime.cli adapter approval check --file adapters/execution-envelope.examples.json --request-id req-20260703-001
```

状态映射：

- request 不存在 -> `needs_input`
- 不需要 approval -> `pass`
- approval `granted` -> `pass`
- approval `pending` -> `needs_approval`
- approval `denied` / `expired` -> `blocked`

输出不包含 input payload。

### `adapter response check`

命令：

```bash
python -m agent_runtime.cli adapter response check --file adapters/execution-envelope.examples.json --request-id req-20260703-002
```

状态映射：

- request 不存在 -> `needs_input`
- response 缺失 -> `needs_input`
- `succeeded` 且 evidence 存在 -> `pass`
- `succeeded` 但无 evidence -> `blocked`
- `blocked` / `failed` / `skipped` -> `blocked`
- `needs_approval` -> `needs_approval`
- `needs_input` -> `needs_input`

输出只包含 response 摘要和计数，不输出 evidence 描述或 raw_ref 值。

## 当前安全边界

- 不执行真实 adapter。
- 不访问网络。
- 不发送消息。
- 不删除文件。
- 不写 `tasks/tasks.jsonl` 或 `tasks/events.jsonl`。

## 推送注意事项

GitHub 直连可能不稳。如需要推送：

1. 先跑 `python -m pytest`、`python -m agent_runtime.cli doctor`、`python tools/public_scan.py`。
2. 直连失败时，再临时启用维护者工具工作区里的 V2Ray 启动脚本。
3. 在项目 checkout 根目录注入 `HTTP_PROXY=http://127.0.0.1:10808` 与 `HTTPS_PROXY=http://127.0.0.1:10808` 后推送。
4. 推送完成后关闭 V2Ray，并确认系统代理恢复。
