### 只读 adapter plan

新增：

- `agent_runtime/adapter_plan.py`
- `tests/test_adapter_plan.py`

命令：

```bash
python -m agent_runtime.cli adapter plan \
  --adapter github-cli \
  --operation git_push \
  --target origin/main
```

行为：

- 复用 `check action` preflight。
- 生成 envelope 草案，不执行 adapter。
- GitHub push 这类需要授权的动作会生成：
  - `adapter_request`
  - `approval_record`
  - `approval_requested` execution event
- 低风险 shell read 只生成 `adapter_request`。
- `--json` 输出完整 envelope。
- 默认人类输出只展示 artifact 摘要。

### 只读 adapter validate

新增：

- `agent_runtime/adapter_validation.py`
- `tests/test_adapter_validate.py`

命令：

```bash
python -m agent_runtime.cli adapter validate --file adapters/execution-envelope.examples.json
```

行为：

- 使用 `adapters/execution-envelope.schema.json` 校验 envelope JSON 文件。
- 输入文件必须在项目根目录内，必须是安全 `.json` 文件。
- 不读取 `.env` / credential。
- 失败输出只给相对路径、schema 错误路径/规则、简短摘要，不回显整条 artifact。
