## 本轮交付

- 新增 `agent_runtime/orchestration_contract.py`，定义 frozen、确定性的 `control-plane/orchestration-contract/v1`。
- 新增 `orchestration contract inspect` / `--json`。
- manifest 覆盖 `stable`、`stable_limited`、`preview`、`unavailable`，包含 command argv、关键 flag、access 和安全 boundary。
- manifest 自描述 `contract_discovery` 能力；unavailable 条目不伪造命令。
- 新增测试冻结字段、排序、分类计数、determinism、no-write、人类可读输出、argparse command path 与 key flag 一致性。
- 既有 orchestration 命令默认输出未改变。
- 第二拍新增 `orchestration contract check`，复用 manifest 做 requirement negotiation。
- 支持重复 `--require`、`--allow-preview` 与 `--max-access`；输出 `control-plane/orchestration-contract-check/v1`。
- unknown/preview 未授权返回 `needs_input`；unavailable/access 超限返回 `blocked`；逐项结果不丢失。
- manifest 新增 `contract_requirement_gate` 自描述 stable 条目，command path 与 key flags 继续由契约测试校验。

## 安全边界

- discovery 命令不读取 ledger、adapter registry、环境变量或凭据；
- 不写文件、不访问网络、不执行 manifest 中的命令；
- 不把 preview 升级为 stable；
- 不新增持久 Run/Report collection 或 orchestration Artifact export。
