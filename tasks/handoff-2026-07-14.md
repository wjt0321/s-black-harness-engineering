## 下一窗口的唯一主任务

按 TDD 完成 Stage 17 第一拍：新增只读、确定性的 Control Panel host handoff descriptor。

建议 CLI：

```bash
python -m agent_runtime.cli orchestration control-panel handoff [--envelope PATH] --json
```

目标不是新增 UI 或 server，而是让本地宿主以机器可读方式发现：

- 当前 `snapshot_id` 与 snapshot schema；
- JSON / HTML 的 media type 与 encoding；
- renderer version 与基于 `snapshot_id + renderer_version` 的 `render_id`；
- snapshot/render 命令入口与 `--envelope` 输入透传规则；
- read-only、no-write、no-network、no-service、no-execution 保证。

Descriptor 不内嵌 HTML、不写文件、不执行其中声明的命令。

## 推荐实施顺序

1. 先在 `tests/test_orchestration_control_panel.py` 写失败测试并确认失败原因正确。
2. 在 `agent_runtime/orchestration_control_panel.py` 复用 `build_control_panel_snapshot()`；不要创建第二套聚合管线。
3. 在 `agent_runtime/cli.py` 添加 `control-panel handoff`，支持 `--json` 与紧凑 human output。
4. 在 `agent_runtime/orchestration_contract.py` 的既有 `control_panel_read` entry 中加入 handoff 命令，不新建重复 capability。
5. 同步 CLI surface/key flag 契约测试与使用文档。
6. 运行全量验证；通过后新增 Stage 17 release notes 并完成本阶段本地提交。

建议重点测试：

- v1 schema/字段顺序/状态；
- `snapshot_id` 与现有 snapshot 命令跨入口一致；
- `render_id` 的确定性与输入变化；
- envelope 缺失、合法、非法三类路径；
- descriptor 不包含 HTML、secret、ledger 原文或 shell 拼接字符串；
- determinism、no-write、no-network/no-execute；
- snapshot/render 默认输出保持兼容；
- contract manifest 与 argparse surface 同步。

## 安全边界

- 不启动 HTTP/server/background process。
- 不访问网络，不加载外部资源。
- 不写 HTML、manifest、draft 或 ledger；第一拍只有 stdout read model。
- 不执行 command 或 adapter。
- 不引入 commit、approval resolve、retry、fallback 或其他 UI 写操作。
- 不读取 `.env`、credential、token、keyring 或密钥文件。
- Secret finding 只输出 rule id、位置与提示，不回显完整匹配值。
