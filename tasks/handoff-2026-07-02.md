## 项目定位

项目根目录：本地 checkout 目录

这是一个长期项目：在 QwenPaw 外部逐步建设轻量的 Agent Runtime / Harness Orchestrator。

目标不是立刻替代 QwenPaw，而是先把这些能力抽象出来：

- Agent 注册表
- 任务路由
- 规则门禁
- 工具适配器
- 任务状态账本
- 交付前验证
- 记忆与文档交接

当前原则：先文档、协议、样例；暂不写真实执行器代码，不启动后台服务，不接管现有生产流程。

## 明天也可以先补的衔接文档

如果不急着进入 Stage 4，可以先写一份小衔接文档：

- `docs/06-policy-task-bridge.md`

内容：policy 命中后如何影响 task 状态。

核心规则：

- 命中 `severity=block` + `action=deny`：任务进入 `blocked`，`blocked_reason=policy_blocked`。
- 命中 `require_user_approval`：任务进入 `blocked`，`blocked_reason=need_user_approval`。
- 命中 `require_secret_scan` 且未扫描：任务进入 `blocked`。
- completion evidence 不足：不得进入 `finished`。
- postflight 失败：回到 `running` 或进入 `blocked`。
