## 当前完整链路

Runtime read-only report 当前闭环为：

```text
task ledger
  -> runtime plan --draft-json
  -> runtime draft validate
  -> runtime draft inspect
  -> adapter validate / adapter inspect
  -> runtime gate check
  -> runtime check-ledger
  -> runtime report
```

各环节职责：

- `runtime plan`：给定 task + adapter action，生成 compact 草案摘要。
- `runtime plan --draft-json`：输出 schema-valid 的 envelope draft 到 stdout。
- `runtime draft validate`：校验 direct envelope 或 `runtime plan --draft-json` wrapper。
- `runtime draft inspect`：输出 draft 摘要，不回显完整 target / input / evidence / raw_ref / decision_ref。
- `adapter validate` / `adapter inspect`：复用 adapter envelope schema 与 consistency 规则。
- `runtime gate check`：聚合 task 状态与 adapter approval/response gate。
- `runtime check-ledger`：审计 task/event ledger 与 adapter envelope 的跨系统一致性。
- `runtime report`：聚合 task snapshot、event summary、envelope summary、gate、ledger audit、blockers 与 next_action。

## 推送记录与注意事项

1. 先跑 `python -m pytest`、`python -m agent_runtime.cli doctor`、`python tools/public_scan.py`。
