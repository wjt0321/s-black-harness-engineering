## Stage 29 输入/输出冻结

输入：

```text
stdin: one control-plane/codex-desktop-snapshot-read/v3 JSON document
max input: 1 MiB
encoding: strict UTF-8
```

候选输出：

```text
control-plane/filtered-snapshot-host-consumer-validation/v1
consumer: codex-desktop-filtered-snapshot-consumer/v1
max stdout: 64 KiB
```

validation result 只输出 base/scope/filter/view ids、checks、value-safe findings、guarantees 与 next_action；不回显 payload、filter values、relative path、rows 或原始输入。

## Identity 冻结

可独立重算：

```text
scope_id  = sha256(canonical({relative_envelope, envelope_content_id}))
filter_id = sha256(canonical(filter))
view_id   = sha256(canonical(filtered_payload_without_view_id))
```

只关联检查：

- base snapshot id；
- envelope content id；
- handoff id。

原因：consumer 不读 envelope、不接收 Stage 24 base payload，也不重跑 handoff producer。

## 固定检查顺序

```text
document_shape
schema_version
reader_status
lifecycle
guarantees
source_scope_identity
filter_identity
representation_links
view_identity
safe_sections
filter_semantics
```

状态/退出码：

- valid ready v3 → `pass` / 0；
- stdin/internal error → `error` / 1；
- unsupported schema/reader、non-ready、unsafe guarantee → `blocked` / 2；
- malformed/duplicate/shape/identity/section/filter mismatch → `validation_failed` / 5。

## Safe summary 边界

只接受：

- runs；
- approvals；
- artifacts；
- request-scoped unavailable reports。

必须验证 counts、matched、section status 与 task/request exact filter semantics。拒绝 project/tasks/registry/automation/raw envelope/descriptor/argv/input/payload refs/raw refs/unknown section。
