## Identity 边界

独立重算：

```text
scope_id  = sha256(canonical({relative_envelope, envelope_content_id}))
filter_id = sha256(canonical(filter))
view_id   = sha256(canonical(filtered_payload_without_view_id))
```

只关联：

- base snapshot id；
- envelope content id；
- source handoff id。

consumer 不读取 envelope、不接收 base payload、不重跑 handoff producer，因此 `pass` 不是真实性或 execution 证明。

## 输出安全边界

输出 source 只包含：

- base snapshot id；
- scope id；
- filter id；
- view id。

不回显 filter values、relative path、summary rows、target、message、raw input、argv 或未知值。

## Stage 30 条件启动

只有具体宿主需要把固定 reader → consumer 管道映射为一次性展示状态时启动。第一拍只做 design gate，冻结：

- 宿主拥有的固定 reader/consumer argv；
- 本地 stdin pipe、timeout、stdout/stderr limits 与取消；
- consumer status/exit 到宿主状态的确定性映射；
- 只在内存中展示 validated safe summaries；
- 不绕过 consumer，不执行 descriptor argv，不自动重试。

若没有具体宿主调用和展示需求，继续维持 Stage 29 冻结，不创建 bridge 空壳。

## Stage 30 明确停止线

不得：

- 修改 Stage 18 consumer contract；
- 让宿主直接信任 reader stdout；
- 自动执行 descriptor argv、candidate command 或 adapter；
- 读取 arbitrary file/URL/project secret；
- query、sort/page、lineage expansion；
- cache/persistence/export、HTML/browser；
- service/network/DB/auth/UI write。
