## 下一步已选路线

下次继续时，从 **ledger 写入前 preflight schema 校验设计** 开始。

选择理由：

- 它不突破当前只读边界。
- 它直接服务 Runtime 主线，为未来安全写入 task ledger 打地基。
- 它比 agent-to-policy 自动映射更底层，也比 public scan 产品化更贴近核心执行链路。

建议目标：

1. 新增一个只读 validation 层，例如 `agent_runtime/task_validation.py`。
2. 支持校验单条 task record 是否符合 `tasks/task.schema.json`。
3. 支持校验单条 event record 是否符合 `tasks/event.schema.json`。
4. 支持 CLI 查询型命令，例如：

```bash
python -m agent_runtime.cli task validate --record-file tasks/tasks.jsonl --schema task
python -m agent_runtime.cli task validate --record-file tasks/events.jsonl --schema event
```

5. 命令只读取文件并输出 validation result；不写入、不修复、不追加 ledger。
6. 失败时输出行号、schema 类型和简短错误摘要；不要回显疑似敏感字段完整值。
7. 补测试覆盖：valid task、invalid task、valid event、invalid event、JSONL 行号报错。
8. 更新 `docs/10-cli-poc-usage.md` 与 `tasks/progress.md`。

实现分工：

- 代码实现优先交给编码 Agent。
- 主控负责需求边界、安全约束、代码审查、测试、doctor、public scan、提交和推送。

备选后续路线：

- agent 到 policy profile 自动映射。
- public scan 产品化设计。
